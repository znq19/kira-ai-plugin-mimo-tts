## 🎯 它做了什么

简单说就一件事：**让 AI 能"用嘴说话"，而且声音你可以自定义。**

```
你：用低沉磁性的男声给我讲个睡前故事
AI：[合成语音消息] "从前有座山，山里有座庙……" 🎵
```

不用选音色编号，不用调参数，**直接像和人说话一样描述声音就行**。

## 🚀 怎么用

### 对 AI 说一句就好

AI 会自动识别你的意图，比如：

| 你说 | AI 做什么 |
|------|-----------|
| 「用温柔的女声给我加油」 | 合成温柔女声的语音发给你 |
| 「用纪录片旁白的语气介绍这个项目」 | 沉稳大气的男声念给你的介绍 |
| 「发个语音跟我说晚安」 | 用默认音色念"晚安" |
| 「念一下这段话」 | 用默认音色合成你引用的内容 |

**不需要记任何命令**，AI 自己会用 `<mimo_tts>` 标签。

### 如果 AI 没领会到

你也可以直接“教”它：

> 「用这种格式发语音：`<mimo_tts voice="活泼可爱的少女声线">你的内容</mimo_tts>`」

## 🔧 工作方式（技术细节）

### 与官方 `<record>` 标签同机制

插件注册了一个叫 `<mimo_tts>` 的消息标签，工作原理和 KiraAI 官方的 `<record>` 一模一样。

```
LLM 输出                                框架处理                          效果
────────────────────────────────────────────────────────────────────────────
<mimo_tts voice="低沉男声">             解析标签 → 调用 API → 合成语音      一条语音消息
大家好</mimo_tts>                        → 插入 Record 元素              直接发出
```

**关键优势**：AI 的决策（要不要发语音）、语音合成、发送 —— **三件事在同一个步骤里完成**。没有额外的工具调用，不会产生垃圾消息，也不影响持续对话功能。

### 对比：为什么不用工具调用？

| 传统工具调用方式 | 本插件的标签方式 |
|------------------|-------------------|
| LLM 先调用 TTS 工具 | LLM 直接输出标签 |
| 工具返回后 LLM 再输出空消息 | 合成和发送一步到位 |
| 多一轮对话步骤，浪费 token | 无额外步骤 |
| 空消息会打断持续对话 | 不影响持续对话 |

## 🎤 音色描述怎么写

**核心原则：你怎么跟人描述声音，就怎么写。**

你可以描述这些维度：

| 维度 | 示例描述 |
|------|----------|
| 性别 | 女声、男声、中性 |
| 年龄 | 少女、青年、中年、成熟 |
| 嗓音 | 温柔、磁性、沙哑、清澈、圆润 |
| 语速 | 语速适中、稍快、缓慢 |
| 情感 | 亲切自然、兴奋活泼、沉稳冷静 |
| 风格 | 像纪录片旁白、像广播剧配音、像深夜电台 |

### 完整示例

```yaml
# 温柔女生（适合日常聊天）
一个年轻温柔的女性声音，语速适中，语调自然亲切

# 沉稳男声（适合解说、旁白）
低沉磁性的男声，缓慢沉稳，像纪录片旁白

# 活泼少女（适合娱乐场景）
活泼俏皮的少女声线，语速稍快，充满活力

# 专业播音（适合正式场合）
沉稳大气的中年男性播音员，字正腔圆

# 深夜电台（适合睡前）
温柔治愈的女声，语速舒缓，像深夜电台主播
```

> 💡 **提示**：描述越具体，合成效果越接近你的预期。但也不用太纠结，AI 会自己帮忙润色。

## 🩹 语音格式自动修复

### 背景：QQ 上的语音有个坑

在 QQ 上，如果你把语音和其他东西（@某人、回复引用）塞在同一条消息里，语音就**发不出去或者显示异常**。

### 插件怎么解决

默认开启的**语音消息格式自动修复**功能，在发送前自动帮你处理：

```
修复前（会出问题）：         修复后（正常显示）：
┌─────────────────┐         ┌─────────────────┐
│ @小明 [语音] 你好  │   →    │ @小明 你好        │  一条文字消息
└─────────────────┘         │ [语音]           │  一条干净语音
                            └─────────────────┘
```

**具体规则**：
- 🎵 每条语音**单独成一条消息**，不混入任何其他东西
- 📎 @ 和回复**自动移到文字消息上**（不会丢）
- ❌ 如果整条消息只有语音没有文字，@ 和回复直接丢弃（保证语音能正常发出）
- ✅ 对官方 `<record>` 标签的语音也生效

> 如果你不需要这个功能，可以在插件设置中关闭「语音消息格式自动修复」开关。

## 📦 安装与配置

### 第一步：获取 API 密钥

1. 打开 [MiMo 平台](https://platform.xiaomimimo.com)
2. 注册 / 登录账号
3. 在 API 管理页面获取你的 **API Key**

### 第二步：安装插件

将整个插件文件夹放入 KiraAI 的 `data/plugins/` 目录：

```
data/
└── plugins/
    └── kira-ai-plugin-mimo-tts/    ← 把整个文件夹放这里
        ├── main.py
        ├── manifest.json
        ├── schema.json
        └── README.md
```

### 第三步：填配置

1. 打开 KiraAI → 插件设置 → 找到「MiMo TTS」
2. 填入 **MiMo API 密钥**
3. （可选）修改默认音色描述
4. （可选）调整语音格式修复开关
5. 保存，搞定！

不用重启，配置实时生效。

## ⚙️ 配置项一览

| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| **MiMo API 密钥** | 密码字段 | （空） | 必填！在 MiMo 平台获取 |
| **默认音色描述** | 多行文本 | 年轻温柔女性声音 | AI 没指定音色时用的默认声音 |
| **语音格式自动修复** | 开关 | 开启 | 自动拆分语音为单条干净消息 |

## ❓ 常见问题

<details>
<summary><b>合成失败了怎么办？</b></summary>

合成失败时，**语音内容会自动降级为文字发送**，确保你的消息不会丢。检查一下：
- API 密钥是否填写正确
- [MiMo 平台](https://platform.xiaomimimo.com) 账户余额是否充足
- 网络是否能访问 `api.xiaomimimo.com`
</details>

<details>
<summary><b>能用英文吗？</b></summary>

可以，MiMo 支持中英文文本合成。音色描述建议用中文写，合成文本中英文都行。
</details>

<details>
<summary><b>普通语音和 MiMo 语音有什么区别？</b></summary>

- **普通语音**（`<record>` 标签）：用 KiraAI 默认的 TTS 引擎，音色固定
- **MiMo 语音**（`<mimo_tts>` 标签）：用小米 MiMo 引擎，**可以自定义音色**

两者可以同时使用，互不冲突。
</details>

<details>
<summary><b>会消耗很多 token 吗？</b></summary>

不会。MiMo TTS 是**独立的语音合成 API**，不占用 LLM 的 token 额度。LLM 只需要输出一个 XML 标签，跟写普通文字差不多。
</details>

<details>
<summary><b>为什么 AI 有时候发了语音但听不到？</b></summary>

如果语音在 QQ 上显示不出来，大概率是格式问题。确保「语音消息格式自动修复」开关是**开启**的（默认就是）。如果还不行，手动让 AI 用单独一条消息发语音即可。
</details>
